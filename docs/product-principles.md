# Product Principles

Every feature added to Cardinal should follow these principles.

## Explain, Don't Display

The application exists to explain information.

Raw data should always support an explanation, not replace it.

## Every Number Answers A Question

Never display a number simply because it exists.

Every value on screen should answer a question that a user naturally asks.

## Progressive Complexity

Simple users should understand the application immediately.

Advanced users should always be able to drill deeper.

Complexity should never be the default.

## Mobile First

Every feature should be designed for a phone before a desktop.

Desktop layouts should enhance the experience, not define it.

## Beautiful Defaults

Cardinal should require little or no configuration before becoming useful.

## Human Language

Prefer sentences over technical terminology.

Example:

Instead of:

Battery Charge
1348W

Prefer:

Your battery is currently charging from excess solar.

## Story Driven

The application should tell the story of a home rather than simply report statistics.

## Trust

Every calculation should be explainable.

Users should always be able to understand where a value originated.
